Production War Stories

Correctness, memory, and observability lessons from real workloads

The Memory War: 9 Months to Fix OOM

The Problem

Early 2025: TPC-DS queries that should take 5 minutes crash with OOM after 90 seconds. Spark's JVM on-heap allocator and Velox's off-heap allocator competed for the same RAM — neither knew about the other.

Root Cause

No unified memory accounting across JVM on-heap and Velox off-heap (via jemalloc). Velox allocated eagerly from OS while JVM GC ran under memory pressure, leading to container OOMs before either allocator could react.

Symptom: Large aggregations (e.g., GROUP BY with 100M+ groups) would OOM at ~60% reported memory usage because Velox's off-heap allocations were invisible to Spark's memory manager.

The Fix & Impact

9 months of engineering: built dynamic memory negotiation between JVM and Velox allocators:

  • Reservation protocol: Velox requests memory quota from Spark's memory manager before allocation
  • GC pressure signals: JVM broadcasts pressure events → triggers Velox spill-to-disk before OOM
  • Graceful degradation: Under contention, Velox spills sorted partitions to SSD with async I/O
↓ 70% OOM incidents (Fabric production telemetry) Gluten 1.3.0 + Fabric Runtime 1.3 (Jan 2025)
→ Upstream Fabric + Gluten community Runtime 1.3 Memory Blog

Fabric-Specific Extensions on Top of OSS Gluten

CategoryFeatureHow It Works
🔐 Security Row-Level Security (RLS) Policy predicates injected at the native scan layer before Velox processes any rows — enforced via JNI bridge before plan translation
Column-Level Security (CLS) Column masking rules applied during Substrait plan generation — masked columns never reach Velox in cleartext
🐍 UDF Support Python UDF Arrow-based columnar transfer via ArrowEvalPython — data stays columnar across JVM↔Python boundary, avoiding Row serialization overhead
Scala UDF JNI-bridged execution with native columnar input — supported UDF classes run without ColumnarToRow transition
⚡ Performance Memory Management Dynamic reservation/release between JVM & Velox allocators — listens to GC pressure signals, triggers native spill before OOM Runtime 1.3 Blog
Spill-to-Disk Optimized off-heap spill paths — sorted spill with compression, async I/O, graceful degradation under pressure instead of OOM
Hash Probe Optimization Hash join probe optimization: improved build-side memory layout for cache locality, explicit prefetch hints for probe-side lookups on large joins
