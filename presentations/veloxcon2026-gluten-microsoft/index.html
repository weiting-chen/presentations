Production War Stories

What broke, what we learned, what we upstreamed

The Memory War: 9 Months to Fix OOM

The Problem

Early 2025: TPC-DS queries that should take 5 minutes crash with OOM after 90 seconds. Spark's JVM on-heap allocator and Velox's off-heap allocator competed for the same RAM — neither knew about the other.

Root Cause

No unified memory accounting across JVM and native heaps. Velox allocated aggressively; Spark GC couldn't reclaim fast enough.

The Fix & Impact

9 months of work: built dynamic memory negotiation between JVM and Velox — smart reservation, release, and spill triggers.

↓ 70% OOM incidents After Gluten 1.3.0 (Jan 2025)
→ Upstream Fabric + Gluten community Runtime 1.3 Memory Blog

More Production Lessons

ChallengeWhat HappenedUpstream Fix
Null Semantics Spark's collect_set includes nulls by default; Velox excluded them → silent data diffs Velox #16416, #16947
Fallback Storms One unsupported expression triggered ColumnarToRow for entire subtree → slower than pure JVM Dynamic filter pushdown #11657, Spark Advisor alerts
Security Boundary Native scan paths must honor Row-Level & Column-Level Security policies JNI bridge security enforcement in Fabric NEE (details in NEE section)
Diagnostics Native SIGSEGV crashes → no JVM stack trace, hard to triage Query trace, Spark Advisor, native crash symbolication
Every row above became an upstream PR or a Fabric-specific hardening. Production is the fastest test suite in open source.

Fabric-Specific Extensions on Top of OSS Gluten

CategoryFeatureHow It Works
🔐 Security Row-Level Security (RLS) Policy predicates injected at the native scan layer before Velox processes any rows — enforced via JNI bridge before plan translation
Column-Level Security (CLS) Column masking rules applied during Substrait plan generation — masked columns never reach Velox in cleartext
🐍 UDF Support Python UDF Arrow-based columnar transfer via ArrowEvalPython — data stays columnar across JVM↔Python boundary, avoiding Row conversion
Scala UDF JNI-bridged execution with native columnar input — supported UDF classes run without ColumnarToRow transition
⚡ Performance Memory Management Dynamic reservation/release between JVM & Velox allocators — listens to GC pressure signals, triggers native spill before OOM Runtime 1.3 Blog
Spill-to-Disk Optimized off-heap spill paths — sorted spill with compression, async I/O, graceful degradation under pressure instead of OOM
Hash Probe Optimization Improved hash join probe: prefetch-friendly memory layout for build-side hash table, reduced cache misses on large joins
